Content below the navigation

This block uses clear: both so it always sits under the floated navigation and action items. The nav itself demonstrates how inline, inline-block and floats interact in a compact header layout.

How the CSS keywords are used here

  • block — .nav-wrap and the section.clearfix-block are block-level by default and occupy full width.
  • inline — the nav list items (li) are set to display:inline so links flow horizontally like words in a sentence.
  • inline-block — the logo and action buttons use display:inline-block so they can set padding/size while still aligning on one line.
  • float — the nav is floated left and the actions floated right to create a classic header layout without flexbox/grid.
  • clear — clear:both on the following block pushes that content below the floated elements.

Resize the browser to see the media query stack the elements for small screens (no flex/grid used — only the requested display modes, float and clear).
